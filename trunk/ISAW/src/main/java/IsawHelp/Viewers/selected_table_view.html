Introduction:


The selected table view submenu provides users with a variety of options for viewing information about selected spectra as a table.



Getting Started:


Launching a Selected Table View


  • Before launching a selected table view, there must be data loaded in the tree.


  • Click on the DataSet of interest and then follow the menus:

View > Selected Table View > Choose a table type

Table Types:


GRX_Y Table


  • Selecting this option will produce a table of the selected spectra with either X or time bin boundary vs. y. It is used when all of the spectra do not have the same X values.



Table View


  • This displays a table with the currently specified fields.


  • Spectra are listed vertically in ascending order.


  • Each X value represents the time bin for that spectrum. Each Y value represents intensity.




Parallel y(x) Table


  • Selecting this option will produce a table of the selected spectra with either X or time bin boundary vs. y.



Table View


  • This displays a table with the currently specified fields.


  • Spectra are listed horizontally in ascending order.


  • Each X value represents the time bin for that spectrum. Each Y value represents intensity.



Special Controls/Readouts


  • Show All Groups - When selected, all available groups will be displayed in the table view. If this option is not checked, only user-selected groups will be shown in the table view.


  • Format – Use this text box to enter Fortran formats such as F5.3, I4, E8.2.


  • Table Readout Area - These readouts provide information about the currently selected cell in the table view including time, the number of counts, the currently selected column/row, channel, d-spacing, Q value, Q space coordinates, energy, wavelength, and integrate. The readouts only respond when the cursor is in the image area.


Special Viewer Menus


Select


Rectangle – This option will mark the spectra corresponding to the selected rectangle from the table.


Instrument Table


  • Selecting this option will produce a table that lists attributes of all spectra. Users can then sort the table and mark selected spectra in a given range.



Table View


  • This displays a table with the currently specified fields.


  • Selected spectra are listed vertically in ascending order.


  • Default fields include group ID, scattering angle, and total count.



Special Controls/Readouts


  • Add Field – Use this option to add a field to the current table. In the new window, click on the field name that you want added to the table.


  • Delete Field – Use this option to delete fields from the current table. To delete a field, first click on a cell in the field you would like to delete, and then press Delete Field.


  • Sort on Field – Use this option to arrange values in a specific field in ascending order. To arrange a field, first click on a cell in the field you would like to arrange, then click Sort on Field.


  • Format – Use this text box to enter Fortran formats such as F5.3, I4, E8.2.


  • Table Readout Area - These readouts provide information about the currently selected cell in the table view including time, the number of counts, the currently selected column/row, channel, d-spacing, Q value, Q space coordinates, energy, wavelength, and integrate. The readouts only respond when the cursor is in the image area.



Special Viewer Menus



Select


Rectangle – This option will mark the spectra corresponding to the selected rectangle from the table.


Clear All – This option clears all selections.


New Attribute – Use this option to add row, column, crate, slot, and input attributes to the table.










Common Viewer Menus



File


Print – This option prints the visible portion of the viewer window as it appears on the screen.


Save As New DataSet – Select this option to save the current view as a new DataSet. The new DataSet will appear in the tree under “Modified.”


Save Image – Select this option to save the visible portion of the viewer window as a JPEG picture file.


Save Object State – This option is not supported by selected table view.


Load Object State – This option is not supported by the selected table view.


Close Viewer – This option closes the image view window.




Edit

Spreadsheet – Use this submenu to copy the selected area of the current table for pasting into a spreadsheet. An integrate function is also available.


Sum – This option is used to sum all counts of either the selected or unselected spectra.


Delete – This option is used to delete all counts of either the selected or unselected spectra.


Clear – Use this option to clear all select flags.


Sort by – This option is used to sort the graph data.




View


Additional View – This option creates a duplicate image view window. This new window can then be changed to a different view type.


Image View – This option changes the current window to an image view window.


3D View – This option changes the current window to a 3D view window.


HKL Slice View – This option changes the current window to a HKL Slice view window.


Contour View – This option changes the current window to a Contour view window.


Scrolled Graph View - This option changes the current window to a Scrolled Graph view window.


Selected Graph View - This option changes the current window to a Selected Graph view window.


Selected Table View – This option changes the current window to a Selected Table View window.


Table Generator – This option changes the current window to a Table Generator window.


Axis Conversions – This option changes the X-axis to a different unit of measurement.




Options


Show Errors – Select this option to display a new field within the table containing errors. The new field name will begin with Er:. This option is unselected by default.


Show Indices – Select this option to display a new field within the table containing indices. The new field name will begin with Ind:.


Link Views – Select this option to link the current view with any additional views of the same data set that also have this option selected. This is on by default so that changes made in one view affects all views of the same data.
