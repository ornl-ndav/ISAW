Java Tutorial for Fortran Programmers


By Chris M. Bouzek

This document is a mixture of a tutorial and a quick reference for Fortran programmers who would like to make the move to Java programming. It does not aim to be a comprehensive guide for developing in Java. Rather, it serves as a gentle entry into the world of cross-platform, object oriented programming in Java.

Essential differences between Fortran and Java

  • In Java, comments start with a "//" rather than a "C".
  • Column position has no relevance in Java, i.e. a statement or comment may begin and end in any column, although the 80 column limit is still observed by convention.
  • Variable names in Java may be 255 characters long, and have roughly the same naming conventions as Fortran (e.g. they may not start with a number).
  • There are no implicit variables in Java, i.e. if you do not give a variable a type when you declare it, the compiler will complain.
  • Although character arrays can be used for text strings, Java has a much more useful String class that we will explore later.
  • goto statements are not allowed in Java.
  • Nearly every Java statement requires a semicolon at the end.
Basic Types


Fortran and Java share many similar data types, although the Fortran complex variable is not supported in Java. Below is a table listing showing some of the parallels between Java and Fortran.

Fortran data type Java data type Description
integer int integer value
real float single precision floating point value
double precision double double precision floating point value
logical boolean true/false value
character char ASCII character value


Declaring Variables


In general, declaring variables in Java is the same as declaring variables in Fortran. Here are some examples of declaring integer values:

int myint1;          //declare an integer named myint1
int myint2 = 255;    //declare an integer named myint2 and initialize it to 255


Note the use of the semicolon at the end of the statement, as it is almost always required after a Java statement. We can also declare a floating point variables:

float myfloat1;         //declare a floating point named myfloat1
float myfloat2 = 2.0f;  //declare a floating point named myfloat2 and initialize it to 2.0


Note here that we used 2.0f rather than 2.0. This is due to a quirk in Java: it does not allow what is called a "narrowing conversion" from double precision to single precision floating point values. To illustrate:

float badfloat = 2.0;    //this will not work
double mydouble = 2.0;   //but this will
